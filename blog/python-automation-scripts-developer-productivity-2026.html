Python Automation Scripts Every Developer Should Know

2026-07-02 · 14 min read · by xbyteid

Python's standard library is a goldmine for automation. Most scripts need zero external dependencies — just the modules that ship with Python. Here are practical automation scripts that save real time in a developer's daily workflow.

Bulk File Renaming

Rename hundreds of files with a pattern — camera photos, downloaded reports, log files:

#!/usr/bin/env python3
"""Rename files matching a pattern in a directory."""

import argparse
from pathlib import Path

def rename_files(directory: Path, pattern: str, replacement: str, dry_run: bool = True):
    """Rename files: replace `pattern` with `replacement` in filenames."""
    count = 0
    for f in sorted(directory.iterdir()):
        if not f.is_file():
            continue
        if pattern not in f.name:
            continue
        new_name = f.name.replace(pattern, replacement)
        new_path = f.parent / new_name
        if dry_run:
            print(f"  {f.name} → {new_name}")
        else:
            f.rename(new_path)
            print(f"  Renamed: {new_name}")
        count += 1
    return count

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="Bulk rename files")
    parser.add_argument("directory", type=Path)
    parser.add_argument("pattern", help="String to find in filenames")
    parser.add_argument("replacement", help="Replacement string")
    parser.add_argument("--execute", action="store_true", help="Actually rename (default: dry run)")
    args = parser.parse_args()

    print(f"{'DRY RUN' if not args.execute else 'EXECUTING'}: Replace '{args.pattern}' with '{args.replacement}'")
    count = rename_files(args.directory, args.pattern, args.replacement, dry_run=not args.execute)
    print(f"\n{count} files {'would be' if not args.execute else ''} renamed")

Usage: python rename.py ./photos "IMG_" "vacation_". The dry-run default prevents accidents — add --execute to actually rename.

Directory Monitor

Watch a directory for new files and process them automatically — convert images, parse CSVs, move downloads:

#!/usr/bin/env python3
"""Watch a directory for new files and process them."""

import time
from pathlib import Path
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler

class NewFileHandler(FileSystemEventHandler):
    def __init__(self, extensions: set[str] | None = None):
        self.extensions = extensions

    def on_created(self, event):
        if event.is_directory:
            return
        path = Path(event.src_path)
        if self.extensions and path.suffix.lower() not in self.extensions:
            return
        # Wait for file to finish writing
        prev_size = -1
        while prev_size != path.stat().st_size:
            prev_size = path.stat().st_size
            time.sleep(0.5)
        self.process(path)

    def process(self, path: Path):
        print(f"New file: {path.name} ({path.stat().st_size} bytes)")
        # Add your processing logic here
        # e.g., convert image, parse CSV, upload to S3

if __name__ == "__main__":
    import sys
    watch_dir = sys.argv[1] if len(sys.argv) > 1 else "."
    handler = NewFileHandler(extensions={".csv", ".json", ".xlsx"})
    observer = Observer()
    observer.schedule(handler, watch_dir, recursive=False)
    observer.start()
    print(f"Watching {watch_dir}...")
    try:
        while True:
            time.sleep(1)
    except KeyboardInterrupt:
        observer.stop()
    observer.join()

Install watchdog with pip install watchdog. The file-size polling loop ensures you don't process a file that's still being written.

Database Backup Script

Automate PostgreSQL or MySQL backups with rotation — keep last 7 daily, 4 weekly:

#!/usr/bin/env python3
"""Automated database backup with rotation."""

import subprocess
import shutil
from datetime import datetime
from pathlib import Path

BACKUP_DIR = Path.home() / "backups" / "db"
KEEP_DAILY = 7
KEEP_WEEKLY = 4

def backup_postgres(db_name: str, output: Path):
    """Dump a PostgreSQL database."""
    result = subprocess.run(
        ["pg_dump", "-Fc", "--no-owner", db_name],
        capture_output=True, check=True
    )
    output.write_bytes(result.stdout)
    size_mb = output.stat().st_size / (1024 * 1024)
    print(f"Backup: {output.name} ({size_mb:.1f} MB)")

def rotate_backups(backup_dir: Path):
    """Keep recent daily and weekly backups, delete older ones."""
    backups = sorted(backup_dir.glob("*.dump"), key=lambda f: f.stat().st_mtime, reverse=True)
    daily = backups[:KEEP_DAILY]
    # Keep one per week beyond daily window
    weekly = backups[KEEP_DAILY::7][:KEEP_WEEKLY]
    keep = set(daily + weekly)
    for f in backups:
        if f not in keep:
            f.unlink()
            print(f"Deleted old backup: {f.name}")

if __name__ == "__main__":
    db_name = "myapp_production"
    today = datetime.now().strftime("%Y-%m-%d")
    backup_dir = BACKUP_DIR / db_name
    backup_dir.mkdir(parents=True, exist_ok=True)
    output = backup_dir / f"{db_name}_{today}.dump"
    backup_postgres(db_name, output)
    rotate_backups(backup_dir)
    print("Backup complete")

Add to cron: 0 2 * * * python3 /path/to/backup.py for daily 2 AM backups. The rotation logic keeps storage bounded without manual cleanup.

CSV Report Generator

Read data from a CSV, compute stats, and write a formatted report:

#!/usr/bin/env python3
"""Generate summary report from CSV data."""

import csv
import json
from pathlib import Path
from collections import Counter

def analyze_csv(filepath: Path) -> dict:
    """Analyze a CSV file and return summary stats."""
    with open(filepath, newline="") as f:
        reader = csv.DictReader(f)
        rows = list(reader)

    if not rows:
        return {"error": "Empty file"}

    columns = reader.fieldnames or []
    numeric_cols = []
    categorical_cols = []

    # Detect column types
    for col in columns:
        try:
            float(rows[0][col])
            numeric_cols.append(col)
        except (ValueError, TypeError):
            categorical_cols.append(col)

    stats = {"total_rows": len(rows), "columns": len(columns)}

    # Numeric summaries
    for col in numeric_cols:
        values = [float(row[col]) for row in rows if row[col]]
        if values:
            stats[col] = {
                "min": min(values),
                "max": max(values),
                "avg": sum(values) / len(values),
                "count": len(values)
            }

    # Top values for categorical columns
    for col in categorical_cols:
        counts = Counter(row[col] for row in rows if row[col])
        stats[col] = {"unique": len(counts), "top_5": counts.most_common(5)}

    return stats

if __name__ == "__main__":
    import sys
    filepath = Path(sys.argv[1])
    report = analyze_csv(filepath)
    print(json.dumps(report, indent=2, default=str))

Uses only stdlib. Run python report.py sales.csv and get instant stats on any CSV — column types, min/max/avg for numbers, top values for categories.

Batch HTTP Status Checker

Check if a list of URLs are responding — useful for monitoring sites, checking link rot, or verifying deployments:

#!/usr/bin/env python3
"""Check HTTP status of multiple URLs concurrently."""

import asyncio
import aiohttp
import sys
from pathlib import Path

async def check_url(session: aiohttp.ClientSession, url: str) -> dict:
    """Check a single URL and return status info."""
    try:
        async with session.get(url, timeout=aiohttp.ClientTimeout(total=10)) as resp:
            return {
                "url": url,
                "status": resp.status,
                "ok": 200 <= resp.status < 400,
                "time_ms": int(resp.headers.get("X-Response-Time", 0))
            }
    except Exception as e:
        return {"url": url, "status": None, "ok": False, "error": str(e)[:80]}

async def check_all(urls: list[str]) -> list[dict]:
    """Check all URLs concurrently."""
    async with aiohttp.ClientSession() as session:
        tasks = [check_url(session, url) for url in urls]
        return await asyncio.gather(*tasks)

if __name__ == "__main__":
    urls_file = Path(sys.argv[1]) if len(sys.argv) > 1 else None
    if urls_file and urls_file.exists():
        urls = [line.strip() for line in urls_file.read_text().splitlines() if line.strip() and not line.startswith("#")]
    else:
        urls = [
            "https://example.com",
            "https://httpbin.org/get",
            "https://httpbin.org/status/404",
        ]

    results = asyncio.run(check_all(urls))

    ok = sum(1 for r in results if r["ok"])
    fail = len(results) - ok
    print(f"\nResults: {ok} OK, {fail} FAILED\n")

    for r in results:
        icon = "✓" if r["ok"] else "✗"
        status = r["status"] or r.get("error", "timeout")
        print(f"  {icon} [{status}] {r['url']}")

Uses asyncio + aiohttp for concurrent checks. Reads URLs from a file (one per line, # for comments) or uses defaults. Install with pip install aiohttp.

Git Repository Cleanup

Clean up merged branches, prune stale remotes, and report repo health across multiple projects:

#!/usr/bin/env python3
"""Clean up git repos: prune branches, gc, report status."""

import subprocess
from pathlib import Path

def git(repo: Path, *args) -> str:
    """Run a git command in the given repo."""
    result = subprocess.run(
        ["git", *args], cwd=repo, capture_output=True, text=True
    )
    return result.stdout.strip()

def cleanup_repo(repo: Path):
    """Clean up a single git repository."""
    print(f"\n{'=' * 50}")
    print(f"Repository: {repo.name}")
    print(f"{'=' * 50}")

    # Fetch and prune
    git(repo, "fetch", "--all", "--prune")
    print("  Fetched and pruned remotes")

    # List merged branches (exclude main/master/develop)
    protected = {"main", "master", "develop", "HEAD"}
    merged = [b.strip() for b in git(repo, "branch", "--merged", "main").splitlines() if b.strip("* ").strip() not in protected]
    if merged:
        print(f"  Merged branches to delete: {len(merged)}")
        for branch in merged:
            branch = branch.strip("* ")
            git(repo, "branch", "-d", branch)
            print(f"    Deleted: {branch}")
    else:
        print("  No merged branches to clean")

    # Garbage collect
    git(repo, "gc", "--auto")
    print("  Ran garbage collection")

    # Report size
    size = subprocess.run(["du", "-sh", str(repo / ".git")], capture_output=True, text=True)
    print(f"  Repo size: {size.stdout.split()[0]}")

if __name__ == "__main__":
    import sys
    base = Path(sys.argv[1]) if len(sys.argv) > 1 else Path.home() / "projects"
    repos = [d for d in sorted(base.iterdir()) if (d / ".git").is_dir()]
    print(f"Found {len(repos)} git repositories in {base}")
    for repo in repos:
        cleanup_repo(repo)
    print(f"\nDone. Cleaned {len(repos)} repositories.")

Run python git-cleanup.py ~/projects and it finds all git repos, prunes stale branches, runs GC, and reports sizes. No external dependencies.

Environment File Sync

Keep .env.example in sync with .env — detect missing variables, flag unused ones:

#!/usr/bin/env python3
"""Sync .env.example with .env — find missing and extra variables."""

from pathlib import Path

def parse_env(path: Path) -> dict[str, str]:
    """Parse a .env file into a dict."""
    env = {}
    for line in path.read_text().splitlines():
        line = line.strip()
        if not line or line.startswith("#"):
            continue
        if "=" in line:
            key, _, value = line.partition("=")
            env[key.strip()] = value.strip().strip('"').strip("'")
    return env

def sync_env(base_dir: Path):
    env = parse_env(base_dir / ".env")
    example = parse_env(base_dir / ".env.example")

    missing = set(env) - set(example)
    extra = set(example) - set(env)

    if missing:
        print("Variables in .env but missing from .env.example:")
        for key in sorted(missing):
            print(f"  + {key}={env[key]}")

    if extra:
        print("Variables in .env.example but missing from .env:")
        for key in sorted(extra):
            print(f"  - {key}={example[key]}")

    if not missing and not extra:
        print("✓ .env and .env.example are in sync")

if __name__ == "__main__":
    import sys
    sync_env(Path(sys.argv[1]) if len(sys.argv) > 1 else Path("."))

Scheduling with Cron and systemd

Run your scripts on schedule. Two options:

Option 1: crontab

# Edit crontab
crontab -e

# Run backup daily at 2 AM
0 2 * * * /usr/bin/python3 /home/user/scripts/backup.py >> /var/log/backup.log 2>&1

# Run git cleanup every Sunday at 6 AM
0 6 * * 0 /usr/bin/python3 /home/user/scripts/git-cleanup.py >> /var/log/git-cleanup.log 2>&1

Option 2: systemd timer (more flexible)

# ~/.config/systemd/user/backup.service
[Unit]
Description=Database backup

[Service]
Type=oneshot
ExecStart=/usr/bin/python3 /home/user/scripts/backup.py

# ~/.config/systemd/user/backup.timer
[Unit]
Description=Daily database backup

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target

Enable with systemctl --user enable --now backup.timer. Systemd timers give you logging, dependency management, and Persistent=true catches up on missed runs.

Key principle: Every script here uses Python's standard library or one small dependency. Before reaching for a framework, check what pathlib, csv, json, subprocess, and asyncio can do. Python ships with batteries included — use them.

The best automation scripts are the ones you actually use. Start with one task that wastes your time weekly, script it, and add it to cron. Over time, you build a personal toolkit that handles the boring stuff while you focus on the interesting problems.
