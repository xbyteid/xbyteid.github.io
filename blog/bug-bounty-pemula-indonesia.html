Bug Bounty untuk Pemula: Mulai dari Nol, Dapetin $1000 Pertama

2026-06-15 · 10 min read
airdrop web3 defi farming multi-wallet

Lo pernah denger orang dapet $50,000 dari nemuin bug di Shopify? Atau $10,000 dari vulnerability di Twitter? Itu bukan hoax. Bug bounty itu real, dan orang Indonesia juga bisa ikut main.

Gue udah hunting sejak 2024. Ini guide no-BS buat lo yang mau mulai.

Apa Itu Bug Bounty?

Sederhananya: perusahaan bayar lo buat nemuin security holes di aplikasi mereka. Lo ga perlu ijazah, ga perlu sertifikat. Cukup laptop, internet, dan kemauan belajar.

Platform utama: HackerOne, Bugcrowd, Intigriti.

Skill yang Lo Butuh

Basic (bulan 1-2)

  • HTTP fundamentals — request/response, headers, cookies, status codes
  • Web basics — HTML, CSS, JavaScript (bisa baca aja dulu)
  • Burp Suite — intercept & modify requests (ini tool #1 lo)
  • OWASP Top 10 — paham semua kategori vulnerability

Intermediate (bulan 3-6)

  • Authentication bypass — OAuth flaws, JWT attacks, session management
  • Injection — SQLi, XSS, SSTI, command injection
  • Business logic — race conditions, privilege escalation, IDOR
  • API testing — REST, GraphQL, parameter fuzzing

Advanced (bulan 6+)

  • Smart contract auditing — Solidity, reentrancy, flash loan attacks
  • Source code review — kalau target open source
  • Chain exploits — SSRF → internal access → RCE

Pilih Target yang Tepat

Ini yang bikin 90% pemula gagal — salah pilih target.

JANGAN: Langsung hunting di Google, Facebook, Apple. Lo cuma bakal dupe laporan yang udah ada.
LAKUKAN: Start dari program yang kurang populer. Cari program dengan bounty tapi sedikit participants.

Cara Cari Target Bagus

  1. HackerOne → sort by bounty, filter participant count rendah
  2. Program baru (< 6 bulan) — belum banyak yang scan
  3. Web3 protocols — bayar gede, less explored
  4. Perusahaan lokal Indonesia — kurang kompetisi

Workflow Hunting

1. Recon (30-60 menit)

  • Subdomain: subfinder, amass
  • Port scan: nmap, naabu
  • Tech fingerprint: whatweb, wappalyzer
  • JS analysis: cari API endpoints, hidden params

2. Scanning (1-2 jam)

  • Directory brute: ffuf, feroxbuster
  • Parameter discovery: paramspider, arjun
  • Vuln scan: nuclei + latest templates

3. Deep Testing (2-4 jam)

  • Manual testing di setiap endpoint
  • Auth flow analysis
  • Authorization testing (IDOR, privesc)
  • Business logic testing

Cara Nulis Report yang Valid

Summary: [One sentence — what's the bug]
Severity: [Critical/High/Medium + CVSS]
Steps to Reproduce:
1. Login sebagai user A
2. Buka /api/users/123/orders
3. Ganti 123 jadi 456
4. Response berisi data user B

Impact: [Apa yang bisa attacker lakukan?]
Fix: [Rekomendasi perbaikan]

Realistic Expectations

  • Bulan 1-3: Mungkin $0. Ini normal.
  • Bulan 3-6: First bounty! $100-500.
  • Bulan 6-12: $500-2000/bulan konsisten.
  • Tahun 2+: $5K-20K/bulan. Top hunters $100K+.

Kesimpulan

Bug bounty itu marathon, bukan sprint. Indonesia masih underrepresented — ini opportunity besar. Mulai dari target kecil, konsisten, terus improve.
